# 클라이언트 - 서버 아키텍쳐
2티어 아키텍쳐 - 리소스가 존재하는 곳(서버)과 리소스를 사용하는 앱(클라이언트)을 분리시킨 것. 클라이언트-서버 아키텍쳐라고 불림.
데이터베이스 - 리소스를 저장하는 공간
- 2티어 아키텍처에 데이터베이스가 추가되면 3티어 아키텍처가 됨
프로토콜 - 통신규약
- 웹 어플리케이션 아키텍처에서 클라이언트와 서버가 서로 HTTP라는 프로토콜을 이용해 메시지를 주고 받고 이를 HTTP 메시지 라고 부름.
